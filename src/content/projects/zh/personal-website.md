# 构建我的个人网站

我决定建立这个网站，是为了拥有一个可以分享我的想法、学习笔记和项目的地方，而不受第三方平台的限制。

## 技术栈

我选择了 **Astro** 作为框架，因为它的“岛屿架构”以及对内容密集型网站的出色性能。它允许我用 Markdown（或 MDX）编写内容，并且默认情况下向客户端发送零 JavaScript。

在样式方面，我选择了 **Vanilla CSS**。虽然像 Tailwind 这样的框架很流行，但我希望保持简单，并完全控制设计。我使用了 CSS 变量来实现主题化和一致的间距。

## 主要功能

### 1. 国际化 (i18n)
该网站支持英语和中文。我实现了一个自定义的 i18n 解决方案，它包括：
- 基于 URL 的内容路由（例如 `/zh/ideas`）。
- 使用中心化的翻译文件来处理 UI 元素。
- 包含一个带有地球图标的语言切换器。

### 2. 内容集合
我使用 Astro 的内容集合（Content Collections）来管理我的“想法”、“学习笔记”和“项目”。这确保了我的 frontmatter 的类型安全，并使查询和过滤内容变得容易。

### 3. 简洁与高级的设计
设计理念是“内容至上”。我使用了清晰的排版系统、大量的留白和微妙的交互，以创造优质的阅读体验。

## 未来计划

我计划添加更多功能，例如：
- 深色模式切换。
- RSS 订阅。
- 为“想法”部分添加更多互动元素。
